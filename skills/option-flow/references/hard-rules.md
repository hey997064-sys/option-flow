# Hard Rules · option-flow 硬规则汇总

> 这些是 skill 的强约束。任一条违反 = 直接重生成。SKILL.md 是主入口，本文件是细则。

## 1. 数据真实性

1.1 **唯一数据源是 `ai_payload` 字典**（字段契约见 `ai-payload-schema.md`）。ai_payload 之外的数字一律不能出现。

1.2 **禁止编造数字**。看到 `None` 就走降级路径，不要自己猜一个"合理值"。

1.3 **禁止"四舍五入归零"**。`pcr_oi = 0.791` 必须写成 `0.791`，不能写成 `0.79` / `约 0.8`。例外：`strike` 按整数/非整数规则显示（240.0 → `$240`；15.5 → `$15.5`）——但仍禁止把非整数 strike 四舍五入归整。

1.4 **必填字段缺失 → 抛错**：`pcr_oi` / `atm_iv_pct` / `oi_distribution` 任一缺失，skill 直接抛错，不出半截报告。

## 2. 单位铁律

字段名后缀自带单位，禁止换算。

| 字段后缀 | 是什么 | 显示规则 | 错例 |
|---|---|---|---|
| `_pct` | 百分比数字 | `43.8` → `43.8%` | ❌ `0.438`（误当小数）|
| `_pp` | 百分点（绝对差） | `5.5` → `+5.5pp` | ❌ `5.5%`（**pp ≠ %**）|
| `_wan` | 万张 | `10.2` → `10.2 万张` | ❌ `102000 张`（×10000）|
| `strike` | USD 数值 | 整数去小数点（`240.0` → `$240`），非整数保留一位小数（`15.5` → `$15.5`）；禁四舍五入归整 | ❌ `240.00 美元` ❌ `$16`（15.5 归整）|
| `pcr_oi` | 比例 | `0.791` → `0.791`（3 位小数）| ❌ `79.1%`（误当百分比）|

**pp ≠ %**（反复强调）：pp 是绝对差（百分点），% 是相对值。`iv_hv_spread_pp = +5.5pp` 表示「IV 比 HV 高 5.5 个百分点」，不是「高 5.5%」。

## 3. 输出风格红线（非合规红线）

**重要**：此 skill **不作为长桥官方发布物**，所以**可以给方向、给交易主线、给操作建议**（破 \$X 看 \$Y、事件前不入场、回踩等位等）。以下红线是为了**内容质量**与**LLM 不擅长的领域**，不是合规约束：

3.1 **禁指明事件类型**（"财报 / FOMC / CPI / 关税 / 数据窗口 / earnings"）——LLM 无日历，会猜错。改成描述现象：
- ❌「市场在为 NVDA 财报定价」
- ✅「5/22 单日 IV 飙至 92%，市场押注当日大幅波动」

3.2 **禁统计行话**（散户看不懂）：
- ❌「第 X 分位 / 标准差 / 相关系数 / 极高 / 极低 / 创新极高 / 创新极低」
- ✅「30 日新低 / 低位 / 中下位 / 中位 / 中上位 / 高位 / 新高」（7 档对照见 SKILL.md PCR 分位描述对照表）
- **特别说明**：「极高 / 极低」会让散户误以为绝对值已极端，但 `pcr_30d_rank_pct` 只反映**相对 30 天历史**的位置。例如 PCR=0.755 + rank=100 应写「30 日新高」（事实），不能写「30 日极高」（强度）

3.3 **禁专业 vol 术语**（散户看不懂）：
- ❌「backwardation / contango / vega / gamma / humped / flat / theta decay」
- ✅「近端凸 / 远端均衡 / 全段紧张 / 期限结构平稳」

3.4 **禁模糊弱化词**（跟"指向性明晰"冲突）：
- ❌「可能 / 或许 / 也许 / 大概 / 似乎 / 看上去 / 大致 / 估计」
- 要么有数字依据给判断，要么直接降级到"无明显信号"

3.5 **禁浮夸而无数据依据**：
- ❌「强烈看多 / 爆炸性 / 颠覆性 / 必涨 / 极致紧张」（无数字支撑时）
- ✅「IV 飙至 92.4%」（有具体数字时"飙""跳"可以用）

3.6 **禁指代非公开信息**：
- ❌「内幕消息 / 知情人士 / 据传 / 业内人士透露」

## 4. 语义约束

4.1 **Wall 方向不可错位**（compute.py 已经保证，LLM 复述时不能写反）：
- Call Wall 必须在现价上方 → `call_wall.distance_pct > 0`
- Put Wall 必须在现价下方 → `put_wall.distance_pct < 0`

**Wall v2 算法（2026-05-24 启用）**：
- 旧算法：现价同侧 ATM ± 10% 内 OI 最大的 strike
- 新算法：现价同侧 ATM ± 10% 内**距 cp 最近**、OI ≥ 3 万张的 strike；若无符合阈值的 fallback 到 OI 最大
- 改动动机：旧算法把 SPY 5/22 Put Wall 选成 $700（距 -6.1% 远端 14.5 万 OI），新算法选 $735（距 -1.4% 近端 4.2 万 OI），对齐散户"近端支撑"直觉
- 远端深度集中点（$700 / $710 / $720）由 `deep_supports` 字段单独暴露

**Max Pain 算法身份（散户常问）**：
- 经典 Max Pain：对每个候选 strike S，计算"假设到期价格为 S 时，所有期权持有者总亏损" = `sum(call_oi * max(0, S-K)) + sum(put_oi * max(0, K-S))`，取 pain 最小的 S
- 我们用 ≤14d 短桶**所有 expiry 的 OI 合并**算 Max Pain（不区分 expiry）
- 散户对账友商可能不一致：友商常用**单 expiry**（如 5/29 weekly）或**只用周末 expiry**，口径不同结果会差几个 strike
- 跟 Wall 一样属于"口径透明"问题，**非 bug**

4.2 **PCR 表述**用 7 档对照（详见 SKILL.md），**禁"第 X 分位"和"极高 / 极低"**：
- `pcr_30d_rank_pct ≤ 5` → 30 日新低（事实陈述）
- 6-20 → 30 日低位
- 21-40 → 30 日中下位
- 41-60 → 30 日中位
- 61-80 → 30 日中上位
- 81-95 → 30 日高位
- ≥ 95 → 30 日新高（事实陈述）

4.3 **§2 含义列 ≤ 15 字**，不重复数值列。

4.4 **§1 末句必须是交易主线**（"破 $X 看 $Y" / "等回踩 $Z 再判断" / "事件前不入场"），不能停在状态描述。

4.5 **§5 策略推荐 strike 约束**：推荐的所有 strike 必须严格来自 ai_payload 字段，**禁止编造任何其他 strike**：
- `key_levels.call_wall.strike` — 上方阻力
- `key_levels.put_wall.strike` — 下方支撑
- `key_levels.max_pain.strike` — 引力中枢

可推荐工具上限 = 3 strike：裸买 Call / Put、Straddle、Strangle、Call Spread、Put Spread、备兑 Call、保护性 Put。**Iron Condor / Iron Butterfly 不推荐**（需 4 strike，超 ai_payload 提供）。

4.6 **§5 不指定具体到期日**：
- ✅「短期到期」「近端到期」「1-2 月到期」「中长期到期」「到期日由读者自选」
- ❌「5/22 到期」「下周到期」「7 月期权」（任何具体日期）
- ❌「财报后到期」「FOMC 后到期」（指明事件类型，违反 3.1）

期限选择是个人持仓周期决策——LLM 没读者风险偏好信息，**不应替读者决定**。

4.7 **OI 数字口径透明**（蝴蝶图 / Wall / Max Pain 引用）：

`oi_distribution` / `call_wall.oi_wan` / `put_wall.oi_wan` 都是 **≤14d 短期所有 expiry 在该 strike 的 OI 合计**——不是单个 expiry 的快照。散户对账友商单 expiry 数字会发现**我们偏高**（合并 5-10 个 expiry vs 友商常显示单 expiry），是口径不同**非 bug**。

- §3 蝴蝶图 header / footer 已经明示口径，LLM **不需要在 §1 / §5 重复标注**
- 但若散户私聊询问"为什么 $750 Call OI 比友商高"，LLM 应回答："本报告的 OI 数字为 ≤14d 短期所有到期日（约 5-10 个 expiry）的 OI 合计，友商通常只显示单个 expiry。两个口径都没错，只是看数据的方式不同。"
- **禁止**为了对齐友商而**人工调整** OI 数字——以 ai_payload 为准。

## 5. 模板段不可重写

LLM **只填变量、不动结构**：
- **§2 数值列模板**：表头、行顺序（PCR → 30D IV → HV → IV-HV → Max Pain → Wall）、bold 标记位置全部固定
- **§3 ASCII 蝴蝶图**：绘制规则见 `ascii-butterfly-template.md`，字符与对齐不能改
- **§4 前 3 行模板**：「近端最紧张 / 近端常态 / 远端均衡」三行的格式与顺序

LLM 可填的开放位（参考 SKILL.md）：
- §1 全段（120-180 字）
- §2 含义列每格（≤ 15 字）
- §4 末句（30-60 字）
- §5 策略推荐（120-180 字 + 固定免责语）— 受 4.5（strike 约束）+ 4.6（不指定到期日）约束

## 6. PCR 时效（broker T+1 机制）

6.1 **PCR 由 broker T+1 提供**，每日 12:00 ET 前后更新次日数据。

6.2 **盘前 / 上午 12 点前跑 → PCR 可能滞后 1 天**。这是机制性的，不是 bug。

6.3 **`data_quality.pcr_lag_days` 字段暴露真实滞后天数**：
- `pcr_lag_days == 0` → 报告头无 ℹ️ banner
- `pcr_lag_days > 0` → 报告头加 ℹ️ banner（文案见 SKILL.md 报告头格式）

6.4 **滞后超 5 天 → fetch 层 stderr WARN**（可能 broker 停发该 symbol PCR feed）。LLM 不需要感知，照常出报告即可。

## 7. 错误降级（SoT 在 SKILL.md）

完整错误降级表见 **SKILL.md「错误降级」段**——本节只说明**抛错 vs 降级**的层级划分：

- **抛错（不出报告）**：`pcr_oi` / `atm_iv_pct` / `oi_distribution` 必填字段任一缺失；非美股 symbol；标的无活跃期权链
- **拒绝路径（不出 §1-§5）**：`data_quality.low_liquidity = true` —— 输出"期权盘子太小"简报（SKILL.md 模板）
- **报告头 banner**：`is_intraday` / `pcr_lag_days > 0` / `reliable = false`（非冷门时）
- **段落降级**：`call_wall` / `put_wall` / `hv_pct` / `iv_hv_spread_pp` / `iv_peak` / `pcr_30d_rank_pct` 任一为 None 时按 SKILL.md 降级表处理

## 机制叙事 · OI 口径护栏（§3 状态读法）

本产品的水位口径是 **持仓量（OI）**，不是做市商 gamma。机制叙事只能用 OI 语言：
> 持仓集中在某 strike → 做市商日常对冲调仓量大 → 调仓时搬动股价 → 形成引力 / 阻力 / 支撑；持仓薄 → 调仓量小 → 引力弱。

**禁用词**（gamma 口径，会误导且我们无此数据）：做市商 gamma / GEX / Gamma Flip / Zero Gamma / vanna / charm / 负 gamma / 正 gamma regime。

**禁止伪造** regime 拐点价位（如"波动率触发位""零 gamma 位"）——我们没有该数据。

`structure_label` 由 compute 指派，LLM **不自创、不改名、不新增类别**；机制方向措辞从简，只说"持仓集中处形成引力/阻力/支撑"，不展开做市商 delta 对冲方向细节（易写反）。
